## What it does

A fictional script that rewards players for bringing specific items to a location. Good template for delivery, turn-in, or NPC interaction jobs.

## Code — with tabs

Most FiveM logic is split between **client** and **server**. Use `<Tabs>` to show them side by side:

<Tabs>
  <TabItem label="Client" icon="seti:lua">
    ```lua title="client.lua" {3,5-7}
    RegisterNetEvent('tq_example:reward', function(amount)
      -- The highlighted lines show the reward flow
      lib.notify({ title = 'Delivery', description = ('+$%d'):format(amount) })

      TriggerServerEvent('tq_example:log', {
        at = GetGameTimer(),
        reward = amount,
      })
    end)
    ```
  </TabItem>
  <TabItem label="Server" icon="seti:lua">
    ```lua title="server.lua"
    RegisterNetEvent('tq_example:log', function(payload)
      local src = source
      print(('[tq_example] player %d rewarded %d'):format(src, payload.reward))
    end)
    ```
  </TabItem>
  <TabItem label="SQL" icon="seti:db">
    ```sql title="install.sql"
    CREATE TABLE IF NOT EXISTS tq_example_log (
      id     INT AUTO_INCREMENT PRIMARY KEY,
      cid    VARCHAR(64) NOT NULL,
      reward INT NOT NULL,
      at     TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );
    ```
  </TabItem>
</Tabs>

## Admonitions

Four built-in admonition styles — use them to flag things readers must not miss.

:::note
Informational. Use for context or background detail.
:::

:::tip[Recommended pattern]
Use for best-practice suggestions. The heading in brackets is optional.
:::

:::caution
Use when something will silently misbehave if ignored (e.g. wrong column type, missing dependency).
:::

:::danger[Do not do this]
Use for actions that corrupt data, break saves, or expose credentials.
:::

## Step-by-step

<Steps>

1. Stop the resource.

   ```sh
   stop tq_example
   ```

2. Edit `config.lua` and change the payout:

   ```lua
   Config.Payout = 500
   ```

3. Start it again.

   ```sh
   ensure tq_example
   ```

</Steps>
